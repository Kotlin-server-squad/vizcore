# Phase 08.5 Plan 01: Live real-app view → IDE-docked layout Summary

**Reflowed the shipped Phase-8 live view into the locked Surface-001 IDE-dock — a new presentational `LiveDockPanel` (metric-tile header strip + 1fr/320px grid with the live list and a single inline amber `LeakList` on the left, a source-panel slot on the right), mounted live-only in `SessionDetails`; metrics are un-buried from the Threads tab and the partial L1 strip + duplicate `SessionMetrics` mount are gone.**

## Accomplishments
- New `LiveDockPanel` IDE-dock shell: locked `border-t-2 border-primary` + `min-h-[200px]` surface, a tiles-only metric-strip header (LivePill + reused `SessionMetrics showLeaks={false}`), a `grid-cols-[1fr_320px]` body, the live list + a single inline amber `LeakList` in the left column, and a source-panel slot (with a muted "Select a coroutine to view its source" placeholder) on the right.
- `SessionMetrics` gained one additive optional `showLeaks?: boolean` (default `true`) guarding its internal "Potential leaks" Card — every existing call site is byte-equivalent (back-compat), the dock passes `false` to keep its header strip tiles-only and avoid a double leak mount.
- Mounted the dock LIVE-ONLY in `SessionDetails` (`isLiveView = !replayActive && !readOnly`); replaced the partial L1 docked-metrics strip and removed the duplicate `SessionMetrics` placement. Replay/shared modes keep the existing tabbed layout with no dock and no added interactivity.
- LeakList sourced from `useSessionMetrics(sessionId, streamEnabled, !readOnly)` — shares the `['session-metrics', sessionId]` React Query key with the strip's internal call, so no extra network fetch; amber/warning only, never danger.
