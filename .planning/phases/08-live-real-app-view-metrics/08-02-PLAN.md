<objective>
Add a per-session backend MetricsProjection over the existing event store/bus, exposed via a new
tenant-scoped GET /api/sessions/{id}/metrics route, delivering RCO-07: active/peak counts,
throughput, dispatcher/thread-pool utilization, and age-threshold leak detection.

Implements D-04 (backend per-session projection, NOT FE-derived, NOT process-global Micrometer),
D-05 (reuse the Phase-2/3 ProjectionService infra; must work under replay), D-06 (metric set), and
D-07 (age-threshold leak heuristic, configurable default ~30s).

Purpose: Make the live real-app session legible at the aggregate level. The projection mirrors
ProjectionService exactly (subscribe in init + rebuildFrom) so it is replay/DB-rehydrate consistent.
Output: MetricsProjection (new), VizSession wiring (field + rehydrate), the /metrics route, the
MetricsResponse/LeakDto DTOs.

Depends on 08-01: dispatcher utilization derives from the dispatcherName/scopeId now propagated by
the synthesizer (08-01 Task 3); without it, real-app sessions carry no dispatcher grouping.
</objective>
