# Phase 8 — UI Design Contract

> Visual and interaction contract for the Live Real-App View + Metrics phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Brownfield phase.** vizcore has a mature, locked frontend design system (HeroUI, pinned `~2.7.11` since Phase 2 / ADR-isolated React Aria line). This phase **extends** that system — it does not introduce a new one. Every token below is the *already-established* project contract, read from `frontend/tailwind.config.js`, `frontend/src/index.css`, and existing components (`StateIndicator`, `DispatcherOverview`, `EmptyState`). New work (live "what's running now" view, metrics panel, leak detection) MUST reuse these tokens — no new color, font, or spacing primitives.

## Spacing Scale

Declared values (Tailwind 4-based scale already in use across the codebase — `gap-2`/`gap-3`/`gap-4`, `py-4`/`py-12`, `p-2`/`p-3`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px (`gap-1`/`p-1`) | Chip clusters (e.g. thread-ID chips), inline icon gaps |
| sm | 8px (`gap-2`/`p-2`) | Compact metric rows, icon+label pairs, dispatcher tile padding |
| md | 16px (`gap-4`/`p-4`) | Default card grid gap, metric-card body padding |
| lg | 24px (`py-6`) | Metrics-panel section padding |
| xl | 32px | Layout gaps between live view and metrics panel |
| 2xl | 48px (`py-12`) | Empty-state vertical padding (matches `EmptyState.tsx`) |
| 3xl | 64px | Page-level spacing |

Exceptions: none. Use HeroUI/Tailwind utility classes (`gap-*`, `p-*`, `space-y-*`) only — do not introduce arbitrary `[Npx]` values for layout.

## Typography

HeroUI/Tailwind text scale already in use. **Exactly 4 sizes, 2 weights** (project convention: `font-normal` 400 body, `font-semibold` 600 emphasis — `font-bold` 700 reserved for single-glance metric *numerals* as in `DispatcherOverview` queue-depth). Declared sizes: 12 / 14 / 18 / 20px (`text-xs` / `text-sm` / `text-lg` / `text-xl`). `text-base` (16px) is intentionally NOT used — metric values share the `text-lg` (18px) token with card titles to keep the count at exactly four.

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Caption / meta | 12px (`text-xs`) | 400 (`font-normal`) | 1.5 — dispatcher descriptions, thread-ID labels, `file:line` attribution, "N more" indicator |
| Body / label | 14px (`text-sm`) | 400 (`font-normal`) | 1.5 — metric labels, list rows, coroutine names |
| Metric value / sub-heading | 18px (`text-lg`) | 600 (`font-semibold`) | 1.4 — active/peak counts, throughput numbers, card titles (`h4`) |
| Panel heading | 20px (`text-xl`) | 600 (`font-semibold`) | 1.2 — metrics-panel / empty-state title (`h3`, matches `EmptyState.tsx`) |

> Inter, line-height 1.5 globally (from `index.css`). Four sizes total — no role may introduce a fifth (notably `text-base`/16px is excluded). Leak-count and peak-count single **numerals only** MAY use `font-bold` (700) for at-a-glance scanning — this is the one established brownfield exception (see `DispatcherOverview` queue-depth `text-lg font-bold`). Do NOT generalize `font-bold` to prose, labels, or headings; it applies to standalone metric numbers exclusively.

## Color

HeroUI semantic theme. Primary = indigo (`#6366f1` light / `#818cf8` dark, from `tailwind.config.js`). The 60/30/10 split maps onto HeroUI's `background` / `default-100` (surfaces) / `primary` (accent) tokens. **Coroutine-state colors are an established semantic contract** (`StateIndicator.tsx`) and MUST be reused as-is — do not invent new state colors.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | HeroUI `background` (app surface, light/dark) | Page background, live-view canvas, panel backgrounds |
| Secondary (30%) | HeroUI `default-100` / `content1` (`Card` surfaces) | Metric cards, dispatcher tiles, coroutine list rows, sidebar/nav |
| Accent (10%) | `primary` indigo `#6366f1` / `#818cf8` | See reserved list below |
| Destructive / alert | `danger` (HeroUI red) | FAILED coroutine state ONLY (per `StateIndicator`) — never a button/CTA color in this phase |

**Accent (`primary`) reserved for:**
- The single primary CTA on the live view ("Enable Live Stream" — existing).
- ACTIVE / WAITING_FOR_CHILDREN coroutine state chips (animated ping dot — existing `StateIndicator` contract).
- The "active count" metric value and the live-stream connected indicator.
- Selected node ring / focus state in the tree/graph (existing).

Accent is NOT used for: every metric card border, every chip, generic body links, or decorative emphasis.

**Coroutine-state semantic colors (LOCKED — reuse `StateIndicator.getStateConfig`):**

| State | HeroUI color | Animated |
|-------|-------------|----------|
| CREATED | `default` | no |
| ACTIVE | `primary` | yes (ping) |
| SUSPENDED | `secondary` | no |
| WAITING_FOR_CHILDREN | `primary` | yes (ping) |
| COMPLETED | `success` | no |
| CANCELLED | `warning` | no |
| FAILED | `danger` | no |

**Metrics / leak surfaces (extend the existing `DispatcherOverview` convention):**
- Healthy / nominal value → `success` (green) — matches queue-depth-zero `FiZap`.
- Elevated / attention (e.g. queue depth > 0, throughput spike) → `warning` (amber) — matches queue-depth-positive `FiAlertCircle`.
- **Leak detection (D-07):** flagged leaked coroutines use `warning` for the count badge + list highlight + graph highlight (a leak is "attention," not a hard failure — `danger` stays reserved for FAILED state). Leak badge uses the `bg-warning/10 text-warning` tile pattern (literal class strings only — see `DispatcherOverview` IN-12 note: never template-construct `bg-${color}/10`).

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | **"Enable Live Stream"** (existing live-view control — unchanged) |
| Live view section title | **"What's running now"** (literal framing from CONTEXT D-08 — active-first) |
| Active-only toggle / completed group | **"Show completed (N)"** — collapses completed coroutines into an expandable aggregate count (D-08) |
| Node-cap indicator | **"N more coroutines"** — shown when rendered nodes hit the cap (D-08); click to expand |
| Empty state heading | **"No live coroutines yet"** |
| Empty state body | **"Start your instrumented app and call `VizcoreClient.start(...)`. Running coroutines will appear here in real time."** (next step: point them at the client lib from Phase 7) |
| Metrics panel title | **"Session metrics"** |
| Metric labels | **"Active"**, **"Peak"**, **"Throughput"** (suffix `/s`), **"Dispatcher utilization"**, **"Potential leaks"** |
| Leak badge | **"N potential leak{s}"** — singular/plural per count (matches `DispatcherOverview` task/tasks pluralization) |
| Leak list row | **"{coroutineName} · alive {duration}"** — e.g. "fetchUser · alive 42s" |
| Leak tooltip | **"Active longer than the leak threshold ({threshold}). May be a stuck or never-completing coroutine."** |
| Throughput empty / zero | **"—"** (em dash, not "0/s") when no events have arrived yet |
| Error state | **"Live stream disconnected. Retrying…"** (the existing `use-event-stream.ts` bounded-backoff retry already drives this — surface it, reconnect is automatic; no manual action required) |
| Destructive confirmation | **None.** This phase has no destructive actions — the live view is read-only observation; disabling the stream is non-destructive and needs no confirmation. |

> Tone: developer-direct, lowercase technical nouns where they name code (`VizcoreClient.start(...)`, `file:line`). Sentence case for headings (matches existing `EmptyState`/`DispatcherOverview` copy). No exclamation marks.
