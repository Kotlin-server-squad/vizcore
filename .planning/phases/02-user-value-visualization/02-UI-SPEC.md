# Phase 2 — UI Design Contract

> Visual and interaction contract for Phase 2 (replay, export, comparison). Generated by gsd-ui-researcher, verified by gsd-ui-checker. Brownfield: derive everything from the existing HeroUI design system — do NOT invent new visual primitives.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none (no shadcn — established brownfield HeroUI system; shadcn gate: N/A, do not initialize) |
| Preset | not applicable |
| Component library | HeroUI `@heroui/react` ^2.6.8 (Button, Chip, Card, Tabs, Slider, Dropdown, Select, Table, Spinner, Tooltip, Modal) |
| Styling | Tailwind CSS 3.4 with `heroui()` plugin (`frontend/tailwind.config.js`), `darkMode: "class"`, HeroUI semantic tokens (`bg-content1`, `text-default-500`, `color="primary"` etc.) |
| Animation | framer-motion ^11.14.4 via `frontend/src/lib/animation-variants.ts` (ADR-011) + `useReplayMotion`/`useAnimatedInView` (ADR-027) |
| Icon library | react-icons — Feather set (`react-icons/fi`) ONLY. New icons this phase: FiDownload, FiVideo, FiImage, FiCode, FiFileText, FiSquare (stop), FiInfo, FiColumns (compare nav) |
| Font | Inter, system-ui fallback (set in `src/index.css`); `font-mono` (Tailwind default stack) for session ids, event counters, elapsed time |

**Rule:** every new surface composes existing HeroUI components with semantic color tokens. No raw hex values in component code; no new component library; no CSS files beyond Tailwind utilities.

## Spacing Scale

Declared values (multiples of 4; matches existing codebase usage):

| Token | Value | Tailwind | Usage |
|-------|-------|----------|-------|
| xs | 4px | `gap-1`, `p-1` | Chip clusters (delta badges), icon gaps |
| sm | 8px | `gap-2`, `p-2` | Button groups, chip rows |
| md-tight | 12px | `gap-3`, `p-3` | Header item clusters, controller rows (established convention — keep) |
| md | 16px | `gap-4`, `p-4`, `space-y-4` | Card body padding, card stacks, ReplayController padding |
| lg | 24px | `space-y-6`, `p-6` | Page-level section stacks (SessionDetails uses `space-y-6`) |
| xl | 32px | `py-8` | Route page vertical padding (`container-custom py-8`) |
| 2xl | 48px | `py-12` | Empty-state vertical padding |

Exceptions:
- Sticky offset `top-16` (64px) for the ReplayController bar — clears the HeroUI Navbar height.
- Icon-only controller buttons stay HeroUI `size="sm"` (32px) as in the existing `ReplayController.tsx`; this is a desktop developer tool, no 44px touch-target requirement.

## Color

All colors are HeroUI theme tokens (light / dark values from the theme in `tailwind.config.js` + HeroUI defaults). Components reference tokens, never hex.

| Role | Token | Value (light / dark) | Usage |
|------|-------|----------------------|-------|
| Dominant (60%) | `background` | #FFFFFF / #000000 | Page background (`bg-background` on Layout) |
| Secondary (30%) | `content1` + `default-100/200` | #FFFFFF / #18181B (+ #F4F4F5 / #27272A) | Cards, sticky ReplayController bar, notice banners, table surfaces |
| Accent (10%) | `primary` | #6366F1 / #818CF8 | Reserved — see list below |
| Destructive / recording | `danger` | #F31260 (HeroUI default) | Recording dot + Stop button, error text, FAILED/CANCELLED states, regression deltas |
| Positive semantic | `success` | #17C964 (HeroUI default) | Connected dot, COMPLETED state, improvement deltas (negative diff) |
| A-only semantic | `warning` | #F5A524 (HeroUI default) | "Only in Session A" chips/badges + node outlines (established in ComparisonView) |
| B-only semantic | `secondary` | #7828C8 / #9353D3 (HeroUI default) | "Only in Session B" chips/badges + node outlines (established in ComparisonView) |

Accent (`primary`) reserved for:
1. Play/Pause button on the ReplayController (the one solid control in the bar — all others `variant="flat"` default)
2. Progress Slider fill + smooth progress bar + timeline playhead line
3. REPLAY mode chip (`color="primary" variant="solid"`)
4. Selection-sync highlight ring on tree-pair nodes (`ring-2 ring-primary`)
5. Active nav/tab/view-toggle states (existing pattern)
6. Confirm-dialog primary action ("Start recording")

Never use `primary` for: notices, badges other than the REPLAY chip, table content, export menu items.

Delta direction convention (locked by existing `DiffStat`): increase = `danger` chip + FiArrowUp, decrease = `success` chip + FiArrowDown, zero = `default` chip + FiMinus.

## Surface Contracts

### 1. Sticky ReplayController bar (D-13)

- Position: `sticky top-16 z-30` wrapper directly below the Session Header card, above the tabs; full content width.
- Surface: keep existing `rounded-lg bg-content1 p-4 shadow-sm`; add `shadow-md` while stuck is NOT required — single shadow tier, no scroll listeners.
- Controls row (left→right): ButtonGroup `size="sm" variant="flat"` — Rewind to start (FiRotateCcw, aria-label "Rewind to start"), Step back (FiSkipBack), Play/Pause (`color="primary"`, FiPlay/FiPause), Step forward (FiSkipForward), Stop (FiSquare, aria-label "Stop replay" — returns to index 0 paused). Right side: speed Dropdown (`Button size="sm" variant="bordered"`, label "{speed}x", options 0.5x/1x/2x/5x).
- Counter: `font-mono text-xs text-default-500`, format "Event {n} / {total}", `min-w-[100px] text-right`.
- Recording state (D-23): when recording, prepend to the controls row a cluster `flex items-center gap-2`: pulsing dot `h-2 w-2 rounded-full bg-danger animate-pulse` + elapsed `font-mono text-xs text-danger` (m:ss) + `Button size="sm" color="danger" variant="flat"` "Stop" (discards partial recording). Playback controls are disabled (`isDisabled`) while recording — the scripted pipeline (D-06) drives them. The recording cluster lives in the controller, never as an overlay on the panel, and is excluded from the captured region (D-08).
- Entry animation: keep existing `initial={{opacity:0,y:10}}` fade-slide; respect `prefers-reduced-motion` per ADR-011.

### 2. Replay mode signals in SessionDetails header (D-15)

- REPLAY chip: `Chip size="sm" color="primary" variant="solid"` text "REPLAY", placed in the header chip cluster next to the coroutines/events chips. No theme-wide tinting.
- New-events badge (D-02): `Chip size="sm" color="warning" variant="dot"` text "{N} new events", clickable (wrap in button, aria-label "Exit replay and jump to live") → exits replay, applies buffered events, forgets cursor (D-04). Hidden when N = 0.
- Replay toggle: `Button size="sm"` with FiPlay/FiRadio pairing follows the existing Live Stream button pattern — `variant="bordered"` label "Replay" when off; `color="primary" variant="flat"` label "Exit Replay" when on. Always visible for any session (D-01).

### 3. Replay panel treatment (D-17, D-18)

- Event-derived panels (tree, graph, thread lanes, timeline, EventsList) render from `visibleEvents`.
- Future events (past cursor): `opacity: 0.3` via `futureOpacity` from `useReplayMotion` (ADR-027) — dimmed, still rendered, never unmounted.
- Timeline playhead: vertical `w-0.5 bg-primary` line at the cursor position, full lane height.
- EventsList: auto-scroll + EventHighlight on current event during play/step; manual scroll pauses follow until next play/step (D-14). No "resume follow" button this phase — play/step resumes it.
- Scrub = snap (no animations, instant re-render); play/step = framer-motion variants per ADR-011/027 at the active speed (D-16, RPLY-03).
- Projection-backed tabs show LiveDataNotice at the top of the tab panel; content unchanged.

### 4. Export feedback (ADR-018)

- Success: toast "Exported {filename}" (e.g. "Exported session-abc123-tree-20260612-1430.png").
- In progress: trigger Button `isLoading` during html2canvas render (1–2s on large panels); ExportMenu items disabled while an export runs.
- Failure: toast (danger) — copy below.
- File naming (discretion, locked here): `{sessionId}-{panel}-{yyyyMMdd-HHmm}.{png|svg|webm|json}`, panel ∈ {tree, graph, threads, timeline, events, replay}.

### 5. Recording pipeline UX (D-05..08, D-23..27)

- One click "Record replay as video" → (if estimate > 120s) RecordConfirmModal → enter replay → seek 0 → red-dot state on controller → auto-play at current speed → auto-stop at last event → toast + download.
- Tab hidden during recording → abort + discard + toast (D-24, copy locked below).
- Capture: active visualization panel only, at 2x resolution (D-27), recording indicator excluded (D-08).

### 6. /compare route (D-09..11, D-19..20)

- Nav: "Compare" item in Layout Navbar, exact existing active-state Button pattern.
- URL: `?a=&b=` search params drive the pickers; changing a Select updates the URL (shareable).
- Page order: h1 → ComparisonView (pickers card, summary delta cards, unique/common tables — unchanged) → SyncedTreePair (only when comparison data present).
- Loading/error/empty states: keep ComparisonView's existing Spinner / danger-bordered Card / empty Card patterns verbatim.
- Unknown session id in `?a=`/`?b=` (strict 404, D-12): show ErrorAlert "Session not found" in place of results — copy below.

## Interaction & Keyboard Contract

ADR-017 shortcut set (locked): `Space` play/pause, `ArrowRight` step forward, `ArrowLeft` step back, `Home` rewind to start, `End` jump to end, `1`–`4` speed (0.5x/1x/2x/5x).

- Shortcuts bind ONLY while replay mode is active; suppressed when focus is in `input`, `textarea`, `select`, or `[contenteditable]` (discretion, locked here).
- `Space` must `preventDefault` to stop page scroll; arrow keys must not fight the Slider — when the Slider itself has focus, native Slider key handling wins.
- All icon-only buttons carry `aria-label` (existing convention — keep).
- Scrubber: HeroUI Slider `size="sm" color="primary"`, `aria-label="Replay progress"`, step 1.
- Reduced motion: all new motion respects `prefers-reduced-motion` (ADR-011); dimming (opacity) is exempt — it is state, not motion.

## Toast System (required by ADR-018 — gap)

No toast system exists in the codebase today. Contract:

- **Primary path:** upgrade `@heroui/react` to ^2.7 (framer-motion ^11.14.4 already satisfies its peer range), mount `<ToastProvider placement="bottom-right" />` in `__root.tsx`, use `addToast({ title, color })` with `color="success"` / `color="danger"`.
- **Fallback** (only if the 2.7 upgrade breaks existing components during planning/execution): `sonner` styled with HeroUI tokens (`bg-content1`, `text-sm`, success/danger accents), same placement and copy.
- Toasts auto-dismiss at 4s; max 3 visible; text at 14px/400.

## Noted Assumptions (headless decisions — flag to planner)

1. **REPLAY chip uses `primary` solid** — only solid-primary chip in the app, making the mode signal unmistakable while staying inside the accent budget. CONTEXT.md D-15 specified placement, not color.
2. **A-only = `warning`, B-only = `secondary`** — derived from the existing ComparisonView chips (`only-in-a` warning / `only-in-b` secondary); D-20 delta badges and tree-node outlines reuse the same mapping for cross-surface consistency.
3. **Toast via HeroUI 2.7 upgrade** — ADR-018 mandates toasts but none exist; upgrade chosen over a new dependency. Planner must verify the 2.7 upgrade against existing component usage (Slider/Dropdown/Tabs APIs are stable across 2.6→2.7).
4. **Stop control semantics** — RPLY-01 requires "stop"; defined as seek-to-0 + paused (distinct from Pause). Rewind-to-start button and Stop are therefore merged visually only if the planner finds the 6-button group crowded; default is both present.
5. **Future-event dim = opacity 0.3** — taken from ADR-027 `futureOpacity`; not re-decided.
6. **File naming** `{sessionId}-{panel}-{yyyyMMdd-HHmm}.{ext}` — exercise of the discretion item in CONTEXT.md.
7. **Shortcut focus-guard rule** (suppress in form fields) — exercise of the discretion item; matches standard log-viewer behavior.
