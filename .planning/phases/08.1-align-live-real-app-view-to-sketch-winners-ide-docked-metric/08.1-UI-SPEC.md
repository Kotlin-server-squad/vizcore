# Phase 08.1 — UI Design Contract

> Visual and interaction contract for the live real-app view alignment phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

> **Phase type: FE-ALIGNMENT (delta, not greenfield).** Phase 8 already shipped the live real-app view + metrics, but the layout diverged from the locked sketch winners (sketches 001-C and 002-B encoded in skill `sketch-findings-vizcore`). This contract specifies the **deltas** that move the existing FE toward those winners. It does NOT redesign the theme — Inter + JetBrains Mono, HeroUI semantic palette, dark-first, radius 12px already exist in the codebase and are reused verbatim. Only the layout (where metrics/leaks live) and source attribution (jump-to-code + live-card source) change.

## Typography

3 roles (the existing FE convention; no new sizes introduced this phase):

| Role | Size | Weight | Line Height | Tailwind |
|------|------|--------|-------------|----------|
| Body / metric numeral | 18px | 700 (bold) | 1.45 | `text-lg font-bold` |
| Label / section heading | 18px heading; 14px label | 600 (semibold) | 1.2 heading / 1.45 label | `text-lg font-semibold` (heading), `text-sm` (label) |
| Mono / `file:line` / id | 12–14px | 400 regular; user-frame **600** | 1.45 | `font-mono text-xs`/`text-sm` |

Weights used: **400 regular** (body, library frames) and **600 semibold** (labels, headings, user frames), plus **700** for metric numerals (already in `SessionMetrics`). User frames render at full opacity + weight 600; library frames render at weight 400 + `text-default-400` (dimmed) — this user-bold/library-dim contrast is the load-bearing signal for stack scannability (sketch 002-B).

Body line-height 1.45 (matches existing theme `--font-sans` line-height); heading 1.2.

## Color

Reuses the **existing locked vizcore theme** (`sketch-findings-vizcore/sources/themes/default.css` + `tailwind.config.js` HeroUI semantic palette). No new colors introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `--bg #0a0a0b` / `--surface #131316` | App background, canvas, dock panel surface |
| Secondary (30%) | `--surface-2 #18181b` / `--surface-3 #1f1f23`, `--border #27272a` | Metric tiles, cards, leak rows, panel chrome, borders |
| Accent (10%) | `--primary #006fee` (HeroUI `primary`) | **Reserved list below** |
| Destructive | `--danger #f31260` (HeroUI `danger`) | FAILED coroutine state ONLY — never used for leaks |

**Accent (`primary`) reserved for:**
1. The **Active** metric numeral + icon (running/active coroutine count) — already `text-primary`.
2. The **LIVE pill** dot/active state and the primary `Active` tile emphasis.
3. The **"Jump to code"** affordance on user-code frames (link color / hover) — the one new interactive accent this phase.
4. Running-state coroutine state dot (existing `StateIndicator` contract).

Accent is NOT applied to all interactive elements. Buttons (Show completed, expand/collapse stack) stay neutral `variant="flat"`/`ghost`. The "N more coroutines" line stays `text-default-500`.

**Leak color contract (LOCKED, do not change):** A potential leak is **`warning` / amber (`#f5a524`)**, never `danger` / red. LeakList already uses literal `bg-warning/10 text-warning border-warning/20`. `danger` is reserved exclusively for the FAILED coroutine state.

## Layout & Interaction Deltas (alignment-specific)

This is the heart of an alignment phase. Each delta = current Phase-8 baseline → locked sketch-winner target.

### Delta L1 — Promote metrics + leaks out of the Threads tab into a docked panel header (sketch 001-C)

| | Current baseline (Phase 8) | Target (locked) |
|---|---|---|
| Location | `<SessionMetrics>` + `<LeakList>` mounted **under the Threads tab** (`SessionDetails.tsx:783`) | A **docked bottom panel** below the existing tree/graph canvas, with a **horizontal metric-tile strip in its header** |
| Tiles | 2-col card grid (`md:grid-cols-2`) buried in a tab | Horizontal strip: **Active · Peak · Throughput /s · Dispatcher util** across the dock header, always visible (not behind a tab) |
| Leaks | Card under Threads tab | Surfaced **inline in the dock**, beside the live list, amber/warning |
| Live signal | none in this region | **LIVE pill + ~150ms poll indicator** in the dock header |

**Contract:**
- Add a docked panel region below the canvas in the live view. **v1: static docked panel (always visible, no collapse/resize)** — collapsible/resizable IntelliJ-style behavior is deferred. Panel uses `--surface` with a `--border` top edge; header strip uses `--surface-2` tiles.
- Tile strip is a single horizontal row (wrap allowed on narrow widths): four tiles `Active / Peak / Throughput /s / Dispatcher utilization`. Reuse the existing `MetricCard`/tile rendering from `SessionMetrics.tsx` but reflow from a 2-col grid to a horizontal strip. Active tile keeps `text-primary`.
- The Threads tab no longer hosts metrics/leaks. `DispatcherOverview` (thread lanes) may remain under Threads; the aggregate **Dispatcher utilization tile** moves to the dock header.
- LIVE/DEMO pill in the header is driven by `streamEnabled` (existing live flag): `streamEnabled === true` → LIVE pill + "~150ms poll"; otherwise DEMO pill.

### Delta L2 — Live "what's running now" cards gain source attribution (sketch 002-B)

| | Current baseline | Target |
|---|---|---|
| Card fields | ID / Job / Scope / Parent only | + **created-at `file:line` chip** (compact) and a **"Jump to code"** affordance |
| Suspension source | rendered as inert `{fileName}:{lineNumber}` text in `CoroutineNodeEnhanced.tsx` / `CoroutineTimelineView.tsx` | user-frame `file:line` is a **clickable jump-to-code target** |

**Contract:**
- Each live coroutine card surfaces a **compact source chip** by default: the first user-code creation frame as `{fileName}:{lineNumber}` in `font-mono`. Data already present: `suspensionPoint.{function,fileName,lineNumber,reason}` and creation-stack attribution — **no backend change**.
- The chip is the collapsed (sketch 002-A) default; clicking the card's source region **expands** to the full stack (sketch 002-B) with two collapsible sections: **"Created at"** and **"Suspended at"**.
- In the expanded stack, **user frames render bold (600) + full opacity + accent `file:line` jump target**; **library frames** (`kotlinx.coroutines`, `kotlin.coroutines`, stdlib) render dimmed (`text-default-400`, weight 400) and are **NOT** jump targets.

### Delta S1 — Jump-to-code affordance

**v1 resolution: copy `file:line` to clipboard.** Clicking a user-code frame's `file:line` copies `{fileName}:{lineNumber}` to the clipboard and shows a success toast (`Copied {fileName}:{lineNumber}`). Rationale: the synthesized event carries only `fileName` + `lineNumber` (no absolute path / no IDE bridge wired), so an `idea://`/`jetbrains://` deep-link cannot resolve reliably this phase. The affordance, hover/cursor styling, and accessible label ship now; the resolution strategy (IDE deep-link) can be swapped behind the same click handler later without changing the visual contract.
- Affordance styling: accent (`text-primary`) on the `file:line`, `hover:underline`, `cursor-pointer`, `role="button"`, `aria-label="Jump to code: {fileName}:{lineNumber}"`, tooltip `Copy file:line`.
- Library frames get none of the above (plain dimmed text).
