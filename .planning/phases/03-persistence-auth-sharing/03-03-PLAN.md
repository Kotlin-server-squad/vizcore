Create `DbRetentionPolicy(db, maxAgeDays=30, maxEventsPerSession=100000, cleanupIntervalMinutes=60, clock=System::currentTimeMillis)` mirroring the existing `RetentionPolicy` lifecycle shape: `start(scope)`, `stop()`, and a directly-callable `cleanup(): Int`. `cleanup()` runs two SQL operations via the Exposed DSL:
1. max-age delete: delete `sessions` where `created_at < now - maxAgeDays` AND `NOT EXISTS (SELECT 1 FROM shares WHERE shares.session_id = sessions.id AND (shares.expires_at IS NULL OR shares.expires_at > now))` (Pitfall 6; the active-share guard — note `expires_at IS NULL` means never-expires = always active). FK cascade removes the deleted sessions' events/shares.
2. per-session event trim: for sessions exceeding the cap, delete the oldest events so `count <= maxEventsPerSession` (a `DELETE ... WHERE seq <= (max_seq - cap)` style bulk delete per RESEARCH Pattern 4 (b) — far cheaper than loading events through the interface).

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| principal to store | authenticated identity drives the tenant filter on every read |
| retention loop to DB | bulk deletes cross into SQL; must honor share liveness |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-09 | Elevation of privilege | ExposedSessionStore reads | mitigate | forUser/tenantId filter applied in the store on EVERY read path (list/get/delete); tested by TenantIsolationTest; ADMIN-only bypass |
| T-03-10 | Tampering | tenancy threaded via global mutable | mitigate | tenantId passed explicitly into createSession (Pitfall 3); no shared mutable principal state |
| T-03-11 | Denial of service (availability) | DbRetentionPolicy cleanup | mitigate | NOT-EXISTS active-share guard prevents deleting sessions backing a live share link (Pitfall 6) |
| T-03-12 | Tampering | retention bulk delete SQL | mitigate | Exposed DSL parameterized deletes; no string-concat SQL |
</threat_model>
