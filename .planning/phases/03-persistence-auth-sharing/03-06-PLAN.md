<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| shared viewer (anonymous) to read-only shell | token in the route is the only credential; UI must expose no mutation path |
| clipboard | share URL written to the OS clipboard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-22 | Elevation of privilege | readOnly SessionDetails | mitigate | readOnly gates OFF all mutation/nav affordances (Run/delete/Compare/Settings/re-share/live-toggle); server also enforces (shared endpoint is read-only) — defense in depth |
| T-03-23 | Information disclosure | shared-link rate-limit feedback | mitigate | 429 surfaces generic "Too many requests" copy; the per-IP limit is enforced server-side (Plan 04, D-12) |
| T-03-24 | Information disclosure | share URL in clipboard/DOM | accept | the URL contains a revocable, expiring token (ADR-019); owner-initiated copy; documented |
| T-03-25 | Spoofing | client-side read-only gating | mitigate | UI gating is convenience; the backend shared endpoint returns only read data and the protected routes still require auth — no trust placed in client gating alone |
</threat_model>
