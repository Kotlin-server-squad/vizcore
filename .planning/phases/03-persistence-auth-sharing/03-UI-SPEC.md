# Phase 3 — UI Design Contract

> Visual and interaction contract for the Phase 3 frontend surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> **Stack is locked, not chosen.** vizcore is a brownfield React 19 app on HeroUI 2.7 + Tailwind 3.4. This phase adds four UI surfaces (`/login`, `/shared/:token`, Share dialog, Manage-shares list); it MUST reuse the existing component library, semantic tokens, and patterns — do NOT introduce shadcn or a second design system. All values below are read from the existing codebase (`tailwind.config.js`, HeroUI default theme, `EmptyState.tsx`, `ErrorAlert.tsx`, `RecordConfirmModal.tsx`, `lib/toast.ts`, `Layout.tsx`) or locked by `03-CONTEXT.md` (D-05..D-13).

---

## Scope — Surfaces This Phase Introduces

| Surface | Trigger | Decision | Notes |
|---------|---------|----------|-------|
| `/login` screen | Any `/api` call returns 401 (auth-on only) | D-05, D-06, D-07 | Username + password → `POST /api/auth/token`; JWT stored in localStorage + in-memory; never shown when auth is off |
| `/shared/:token` viewer shell | Public link open (token = credential) | D-09, D-10 | Standalone read-only shell: tree/graph/timeline/thread-lanes + replay + export; NO nav chrome, NO Run/Compare/Settings/mutation |
| Share-creation dialog | "Share" action on a session | D-11 | HeroUI `Modal` (`size="sm"`), expiry picker (1d/7d/30d/never) → `POST /api/sessions/:id/share` → copy-link |
| Manage-shares list | "Manage shares" on owning session | D-13 | Rows: token (short), expiry, hits (`access_count`), last accessed, Revoke (`DELETE /api/sessions/:id/shares/:token`) |

**Not UI (do not render):** the "persistence enabled without auth" notice is a backend startup log only (D-04b) — it must never appear in the UI.

## Typography

HeroUI default scale; declared roles below cover every Phase-3 surface (matches existing `text-xl font-semibold` heading + `text-sm`/`text-default-500` body usage):

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body | 16px (`text-base`) | regular (400) | 1.5 |
| Label | 14px (`text-sm`) | regular (400) | 1.5 |
| Heading | 20px (`text-xl`) | semibold (600) | 1.2 |
| Display | 24px (`text-2xl`) | semibold (600) | 1.2 |

Only two weights: regular (400) and semibold (600) — matches existing `font-semibold`/`font-bold` brand-only usage. Use `text-sm` for table cell content in the Manage-shares list and form helper text; use `text-xl`/`text-2xl` semibold for the `/login` and Share-dialog headings.

## Color

Inherited from HeroUI semantic tokens (primary indigo `#6366f1` light / `#818cf8` dark, per `tailwind.config.js`). The 60/30/10 split maps to HeroUI roles already in use:

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `background` / `default-50` | Page background, viewer canvas, login page surface |
| Secondary (30%) | `content1` / card surface (HeroUI `Card`) | Login card, Share dialog, Manage-shares table container |
| Accent (10%) | `primary` (`#6366f1` / `#818cf8`) | See reserved list below |
| Destructive | `danger` | Revoke button + revoke-confirm action only |

Accent (`primary`) reserved for: the `/login` submit button ("Sign in"), the Share dialog's "Create link" / "Copy link" primary action, and active nav state (existing convention). NOT used for body text, table rows, links-in-prose, or the read-only viewer chrome. The `/shared` shell shows the visualization in its native panel colors with no added accent.

Status colors (existing convention, reuse — do not invent new): `success` for copy-link confirmation toast, `danger` for login-failure `ErrorAlert` and revoke, `default-500` for muted metadata (expiry, last-accessed timestamps).

## Interaction Contracts

| Behavior | Contract | Source |
|----------|----------|--------|
| 401 → login | Any `/api` 401 redirects to `/login`; on success store JWT (localStorage + in-memory source of truth), attach `Authorization: Bearer <jwt>`, return user to the route they came from | D-05, D-06 |
| Auth-off invisibility | When auth is unconfigured no 401 ever fires → `/login` never renders; no UI change vs today | D-04a, D-07 |
| Share dialog flow | Open → pick expiry → `Create link` → field populates with returned URL → `Copy link` → `Link copied` toast; dialog is dismissible at any step | D-11 |
| Copy link | Use `navigator.clipboard.writeText`; on success `toastSuccess('Link copied')`, on failure `toastError` and leave the URL selectable for manual copy | D-11 |
| Revoke | `Revoke` opens the destructive confirm modal → `Revoke link` → `DELETE` → row removed + success toast; subsequent token access returns 410/404 (backend) | D-13, ADR-019 |
| Shared shell | Reuse `SessionDetails` + panels behind a `readOnly` prop / shared-route wrapper — do NOT fork; hide Run/record/Compare/Settings/nav; keep replay (play/scrub/speed) + export (PNG/SVG/WebM/JSON) | D-09, D-10 |
