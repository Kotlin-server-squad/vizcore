<objective>
Deliver revocable, expiring, rate-limited share tokens for read-only session access (SHAR-01, SHAR-02, D-11/D-12/D-13). Rewrite the in-memory `ShareTokenService` prototype into a DB-backed service matching the ADR-019 shape (createdBy, accessCount, lastAccessedAt, nullable expiry), expose the four endpoints, and rate-limit the public read per-IP.

Purpose: teammates can open a session via a link without an account, with owner control over expiry and revocation, protected against scraping/brute force.
Output: `share/` package (ShareService, ShareRoutes, ShareDtos), the public `GET /api/shared/{token}` registered outside auth, Ktor RateLimit install, and `ShareRoutesTest` + `RateLimitTest`.
</objective>

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| anonymous viewer to public shared read | token in the URL path is the only credential |
| owner to share-management | authenticated owner mints/lists/revokes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-13 | Information disclosure / DoS | GET /api/shared/{token} brute force / scraping | mitigate | UUID v4 tokens (122-bit entropy) + per-IP RateLimit -> 429 (D-12); revocable + expiring |
| T-03-14 | Tampering | ShareService Exposed queries | mitigate | Exposed DSL parameterizes token/sessionId; no string-concat SQL |
| T-03-15 | Information disclosure | share-token-in-URL leakage | accept | tokens are revocable + expiring; avoid logging full share URLs; documented (ADR-019 model) |
| T-03-16 | Elevation of privilege | owner share routes | mitigate | mint/list/revoke wrapped in authenticatedApi; createdBy from principal; revoke scoped to {sessionId, token} |
| T-03-17 | Denial of service | retention deleting shared session | mitigate | handled by Plan 03 active-share guard (Pitfall 6) — shares persist while live |
| T-03-SC | Tampering | ktor-server-rate-limit install | mitigate | OK in RESEARCH Package Legitimacy Audit (Maven Central, JetBrains first-party); no [ASSUMED]/[SUS] |
