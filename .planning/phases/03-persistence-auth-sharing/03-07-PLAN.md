<objective>
Close the three confirmed Phase 3 verification gaps that break the central
multi-tenant security invariant (AUTH-04) and its test coverage (AUTH-05):

1. CR-01 — five authenticated session-bound handlers (/events, /hierarchy,
   /threads, /coroutines/{id}/timeline, SSE /stream) resolve the session through
   the UNSCOPED `SessionManager.getSession`, leaking any tenant's session content
   to any authenticated tenant.
2. CR-02 — share-owner routes (mint/list/revoke) check existence only (also
   unscoped) and never enforce ownership, letting any authenticated user mint a
   public share for another tenant's session and list/revoke others' shares.
3. AUTH-05 coverage gap — `TenantIsolationTest` exercises only the store API and
   top-level CRUD, which is why the green suite missed CR-01/CR-02.

Purpose: make the phase goal ("safe to deploy for multiple users — tenant
isolation") actually true. These gaps are NOT deferred to a later phase.
Output: tenant-scoped reads on every session-bound route, ownership-enforced
share routes, and e2e tests that guard the isolation invariant on those paths.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| authenticated client → session-bound API | A valid JWT/API-key identifies tenant X but the request targets a session id that may belong to tenant Y (untrusted session id parameter). |
| authenticated client → share-owner API | A valid principal targets a session/share it may not own (untrusted session id + token). |
| public token holder → /api/shared/{token} | Out of scope here (token IS the credential; unchanged). |
