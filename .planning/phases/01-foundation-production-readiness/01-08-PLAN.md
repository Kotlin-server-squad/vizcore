<objective>
Close Verification Gap 3 (CR-03): core `VizScope` declares `override val coroutineContext: CoroutineContext = context + CoroutineName(...)` with default `context = EmptyCoroutineContext`, so there is NO Job in the context. Consequences: `cancel()` and `cancelAndJoin()` do `coroutineContext[Job]?.cancel()` against a null Job — silent no-ops; every root `vizLaunch`/`vizAsync` is unparented (GlobalScope-equivalent), violating the project's "never GlobalScope, use structured concurrency" rule; and closing a session cannot stop running scenarios. Add a Job to the context so cancellation is functional, parent it to the session scope so session close stops scenarios, and add a regression test proving `cancelAndJoin()` stops a running coroutine.

Purpose: This restores structured concurrency to the authoritative core VizScope. CR-03 is review-driven and not in the original ROADMAP success criteria; it is tagged to FND-01 (single authoritative core class) because the fix lands in the de-forked core VizScope.
Output: a Job inserted into VizScope.coroutineContext; a new VizScopeCancellationTest proving cancelAndJoin stops a running coroutine.

SEQUENCING: This plan depends on 01-07 (wrappers fork deletion). After 01-07, only the coroutine-viz-core copy of VizScope.kt exists, so this edit lands in exactly one file. Do NOT run this before 01-07, or a divergent fork VizScope (without the Job) could still be loaded by classloader ordering and mask the fix.
</objective>
