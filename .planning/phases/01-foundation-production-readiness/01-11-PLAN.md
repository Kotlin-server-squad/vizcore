<objective>
Close the Jobs-tab gap: the backend serializes events with kotlinx-serialization's DEFAULT discriminator field `type` (the `kind` interface property is a computed getter with no backing field, so it is NOT in the JSON). FE types, filters, and category detection compare against `event.kind`. Where an event reaches a consumer un-normalized (or normalization drops kind), `event.kind === 'JobStateChanged'` is undefined and the Jobs tab shows Jobs (0) despite JobStateChanged events being present in the store.

Goal: guarantee a single, consistent discriminator (`kind`) reaches every FE consumer for BOTH the REST-derived path (getSessionEvents → normalizeEvents) and the SSE-derived path (use-event-stream), and prove it with a test that uses the REAL raw backend payload shape rather than pre-normalized fixtures.

Purpose: a session with JobStateChanged events shows the Jobs tab with a correct count.
Output: hardened normalizeEvent + SSE kind assignment, an audited jobStates derivation in SessionDetails, and a discriminator test exercising the raw `{type: ...}` payload end to end.
</objective>
