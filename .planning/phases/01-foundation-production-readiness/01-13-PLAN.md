<objective>
Close the two remaining BLOCKER gaps from 01-VERIFICATION.md (score 10/12) that were introduced by plan 01-12's polling-storm fix. Both are live-stream freshness regressions confined to the same three frontend files:

1. **CR-01 — Threads tab frozen during live streaming.** `useThreadActivity` sets `refetchInterval: false` when `isLive=true` (use-thread-activity.ts line 30) on the premise that SSE-driven cache invalidations refresh the Threads view. That premise is false: the debounced `invalidateQueries` in use-event-stream.ts targets `['sessions', sessionId]` only — the `['thread-activity', sessionId]` key is never invalidated anywhere in the repo (repo-wide grep returns zero hits). The Threads tab freezes for the entire live session.

2. **CR-02 — Trailing-edge debounce starvation under sustained event streams.** Both new debounces (400ms `INVALIDATION_DEBOUNCE_MS` in use-event-stream.ts line 96, 500ms inline in SessionDetails.tsx line 93) are pure trailing-edge with no max-wait cap. Under a sustained stream whose inter-event gap is below the debounce window, each new event perpetually resets the timer, so the session snapshot, job states, and the completion-aware scenario button never refresh during the stream — precisely during the long-running scenarios this tool visualizes live.

Purpose: the live session page stays fresh during streaming — the Threads tab updates and the session snapshot refreshes at a bounded cadence — while preserving the burst-coalescing behavior that eliminated the original ~88-requests-in-3s polling storm.
Output: max-wait-capped debounces in both use-event-stream.ts and SessionDetails.tsx, a thread-activity invalidation wired into the SSE debounce block (plus a slow fallback poll restored in use-thread-activity.ts as defense-in-depth), and regression tests proving both the max-wait flush and the thread-activity invalidation fire.

Depends on 01-12: this plan edits the exact files 01-12 modified and builds directly on its debounce implementation. Run after 01-12.
</objective>
