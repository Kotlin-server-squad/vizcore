<objective>
Close UAT gap 2 (01-UAT.md, severity major): "SSE connects on first load of a freshly created session (gallery → Run flow)" — currently failed. On a session with zero stored events the backend SSE handler writes no bytes (headers never flush) until the first event arrives; curl observes HTTP 000, and through the Vite dev proxy the browser's EventSource request dies with HTTP 500. Per the EventSource spec a non-200 response is FATAL — no auto-reconnect — so the live view is permanently dead until reload: badge stuck "Connecting…", event count frozen at the last REST fetch, Run button stuck "Scenario Running".

Fix both sides per the UAT `missing` items:
1. Backend: send an immediate SSE comment frame on stream open so status/headers flush even when the replay loop has nothing to write (artifact: SessionRoutes.kt:176 sse route).
2. Frontend: handle fatal EventSource errors with bounded exponential-backoff retry instead of giving up forever (artifact: use-event-stream.ts). Because the backend replays the FULL history on every new connection (REVIEW WR-01), retry without dedup would visibly duplicate every event in the live view — so the retry mechanism includes a seq high-water-mark guard. Without it, the retry would corrupt the very view it exists to recover.

Purpose: the primary gallery → Run user flow gets a live stream on first load, and transient stream failures self-heal.
Output: backend heartbeat comment + zero-event flush test; frontend reconnect with backoff, replay dedup, and value-asserting fake-timer tests.
</objective>
