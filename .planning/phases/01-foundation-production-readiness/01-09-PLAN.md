<objective>
Close gap 1: backend instrumentation emits JobStateChanged AFTER a coroutine's terminal lifecycle event, which makes the app's own validator report NoEventsAfterTerminal failures (3× on the Exception Handling scenario: e.g. CoroutineFailed@seq=22 followed by JobStateChanged@seq=23).

Decision (per UAT gap 1 `missing`): emit JobStateChanged BEFORE the terminal lifecycle event so the terminal event always carries the highest seq for its coroutine. This aligns instrumentation with the existing NoEventsAfterTerminalRule rather than weakening the rule — keeping validation honest for user code too.

Purpose: Run Validation on a normal scenario must pass without the app flagging its own event stream.
Output: Reordered emission in VizScope.vizLaunch completion handler + a regression test that runs the validator over an instrumented failing-child scenario and asserts zero NoEventsAfterTerminal findings.
</objective>
