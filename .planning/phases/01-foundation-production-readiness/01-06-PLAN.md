## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-06-01 | Denial of Service | sseClientsGauge | accept | The gauge is an observability counter only; an attacker opening many SSE streams is bounded by the same connection limits as any SSE consumer (out of scope here; rate limiting tracked as PERF-04 in Phase 4). The decrement-in-finally guarantees the gauge cannot leak upward on disconnect. |
| T-01-06-02 | Tampering | gauge counter accuracy | mitigate | Placing `decrementAndGet()` in the `finally` block ensures every connection path (normal completion, client disconnect via CancellationException, server error) decrements exactly once, preventing a permanently inflated gauge that would mislead operators. |
| T-01-06-SC | Tampering | gradle test dep (ktor-client-sse) | mitigate | ktor-client-sse is a first-party Ktor artifact from the same group (`io.ktor`) and version catalog already used by ktor-client-content-negotiation; no new untrusted publisher is introduced. |
